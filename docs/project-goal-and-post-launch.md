# NexusM&A プロジェクトの目標と公開後運用

## プロジェクトの目的

NexusM&Aサイトの目的は、電話営業や紹介依存に偏らず、M&Aや事業承継をまだ具体的に考えていない中小企業経営者との接点をつくること。

特に、いきなり「会社を売りませんか」と迫るのではなく、経営者が自分ごととして考えやすい入口を用意する。

中心導線:

```text
社長不在90日診断
↓
経営課題・承継課題の自己認識
↓
無料相談フォーム
↓
メール中心の初回接点
↓
M&A・事業承継・第三者承継・社内承継などの選択肢整理
```

## 事業上の目標

- 譲渡候補先、譲受候補先、通常問い合わせの自然流入を増やす
- 営業電話に頼らず、問い合わせフォームと匿名診断から相談を生む
- `sasa-eru.com` をNexusM&Aの本命ドメインとして育てる
- 中小M&Aガイドライン遵守、運営者情報、料金説明を明示し、信頼性を担保する
- すぐにM&Aを考えていない経営者にも「まず診断してみる」行動を促す

## このサイトで重視するユーザー

### 売り手・譲渡候補

- 後継者不在だが、M&Aまではまだ考えきれていない経営者
- 従業員、取引先、社名、経営者保証を心配している経営者
- 電話営業に抵抗があり、まずはメールや匿名診断から始めたい人

### 買い手・譲受候補

- 成長戦略として中小M&Aを検討している企業
- 希望業種、地域、規模を登録したい企業
- まだ具体案件ではなく、方針整理から入りたい企業

### その他

- 提携先、士業、金融機関、メディア、採用・協業の問い合わせ

## 公開後に見る指標

### Search Console

週1回確認する。

- 検索結果の表示回数
- クリック数
- クリック率
- 平均掲載順位
- 検索クエリ
- インデックス登録ページ数
- サイトマップの取得状況
- モバイル表示の問題

最初の1〜2週間はデータが少なくても正常。

### 問い合わせ

月次で確認する。

- 問い合わせ件数
- 診断経由の問い合わせ件数
- 売り手、買い手、その他の比率
- メール返信率
- 初回面談化率
- 有効相談率

### コンテンツ

月次で確認する。

- よく見られているページ
- Search Consoleで表示され始めた検索語句
- 問い合わせにつながったページ
- 追加すべき記事テーマ

## 次にやるべき改善

### 1. Search Console初期確認

サイトマップ送信後、1〜3日後に確認する。

- サイトマップが成功になっているか
- `https://sasa-eru.com/` がインデックス登録対象になっているか
- クロールエラーが出ていないか

### 2. 問い合わせフォームの本番再確認

独自ドメイン化後に再度確認する。

- 通常問い合わせ
- 売り手相談
- 買い手相談
- 診断結果からの問い合わせ引き継ぎ

### 3. コラム追加

検索流入を狙う記事を追加する。

優先テーマ:

- 後継者不在 会社 どうする
- 事業承継 M&A メリット デメリット
- 中小企業 M&A 相談前に準備すること
- 経営者保証 M&A
- 社長しか分からない会社の承継リスク
- M&A 電話営業が嫌な経営者向けの相談方法

### 4. 通常URL化の検討

現在は `#/company` のようなハッシュルーティング。
公開優先では問題ないが、SEOを強めるなら将来的に通常URLへ移行する。

例:

```text
現在: https://sasa-eru.com/#/company
理想: https://sasa-eru.com/company/
```

## 判断基準

このサイトの改善では、次の基準を優先する。

- 経営者が警戒せずに読み進められるか
- 電話営業っぽさを消せているか
- 相談前の不安が減るか
- ガイドライン遵守・運営者情報・料金が明確か
- 診断から問い合わせへの導線が自然か
- 実在実績に見える誤解や誇大表現がないか
